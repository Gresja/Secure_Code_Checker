Issue #01

WEAK_HASHING_ALGORITHM

HIGH

Line: 5 | File: samples/hashing_vulnerable.py

Weak hashing algorithm detected (MD5 or SHA1). These are vulnerable to collision attacks. Use SHA-256 or SHA-3 instead: hashlib.sha256().

Why this matters

MD5 is cryptographically weak and can be abused via collisions in integrity checks.

Recommended fix

For passwords, use `bcrypt` or `argon2` (not general-purpose hashes). For file/message integrity, prefer `hashlib.sha256`.

Code context return hashlib.md5(password.encode()).hexdigest() # weak!
